<p>React has become the go-to library for building user interfaces. Let's explore some best practices and patterns that will help you write better React code.</p>
<p><img src="/images/react-component-hierarchy.png" alt="React Component Hierarchy"></p>
<h2>1. Functional Components</h2>
<p>Modern React emphasizes the use of functional components with hooks:</p>
<pre><code class="language-jsx">// Good Practice
const UserProfile = ({ user }) => {
  const [isEditing, setIsEditing] = useState(false);

  return (
    &#x3C;div>
      &#x3C;h2>{user.name}&#x3C;/h2>
      {isEditing ? (
        &#x3C;EditForm user={user} />
      ) : (
        &#x3C;DisplayInfo user={user} />
      )}
    &#x3C;/div>
  );
};
</code></pre>
<h2>2. Custom Hooks</h2>
<p>Extract reusable logic into custom hooks:</p>
<pre><code class="language-javascript">const useLocalStorage = (key, initialValue) => {
  const [storedValue, setStoredValue] = useState(() => {
    try {
      const item = window.localStorage.getItem(key);
      return item ? JSON.parse(item) : initialValue;
    } catch (error) {
      return initialValue;
    }
  });

  return [storedValue, setStoredValue];
};
</code></pre>
<h2>3. Performance Optimization</h2>
<h3>Memoization</h3>
<p>Use React.memo and useMemo wisely:</p>
<pre><code class="language-jsx">const ExpensiveComponent = React.memo(({ data }) => {
  return (
    // Expensive rendering logic
  );
});
</code></pre>
<h3>useCallback</h3>
<p>Optimize callback functions:</p>
<pre><code class="language-jsx">const handleSubmit = useCallback((data) => {
  // Handle form submission
}, [/* dependencies */]);
</code></pre>
<h2>4. State Management</h2>
<p>Choose the right state management approach:</p>
<ul>
<li>Use local state for simple components</li>
<li>Context API for shared state</li>
<li>Redux/Zustand for complex applications</li>
</ul>
<pre><code class="language-jsx">const AppContext = createContext();

const AppProvider = ({ children }) => {
  const [theme, setTheme] = useState('light');
  
  return (
    &#x3C;AppContext.Provider value={{ theme, setTheme }}>
      {children}
    &#x3C;/AppContext.Provider>
  );
};
</code></pre>
<h2>5. Error Boundaries</h2>
<p>Implement error boundaries to handle runtime errors:</p>
<pre><code class="language-jsx">class ErrorBoundary extends React.Component {
  state = { hasError: false };

  static getDerivedStateFromError(error) {
    return { hasError: true };
  }

  render() {
    if (this.state.hasError) {
      return &#x3C;h1>Something went wrong.&#x3C;/h1>;
    }
    return this.props.children;
  }
}
</code></pre>
